# RailGuard Timed Demo Cheat Sheet

This guide outlines how to perform a 2-minute live demo of the RailGuard system. The demo highlights the autonomous loop, linear prediction engine, auto work order generation, and Gemini AI integration.

---

## ⏱️ Timeline & Script

### Phase 1: Autonomous Baseline (0:00 - 0:20)
1. **Boot App:** Open the dashboard.
2. **Observe Grid:** 100 cells are healthy (green). 
3. **Observe Activity Log:** Terminal scrolling shows:
   `[MONITOR] SCAN Cycle #X: Scanning 100 segments...`
   *Point out: "The system is autonomous. It monitors 100 segments on a 10-second interval without any human clicks."*

### Phase 2: Degradation & Prediction (0:20 - 0:50)
1. **Trigger Scenario:** 
   - Under **Simulator Panel -> Scenarios**, select **"Degrade SEG-042 (Critical)"** and click **Run**.
2. **Observe Transitions:**
   - SEG-042 instantly highlights red (Critical status).
   - The **Activity Log** prints:
     `[DETECTION] ANOMALY SEG-042 → critical (risk: X)`
     `[DISPATCH] WORK_ORDER Auto-generated WO-SEG-042-001 (priority: urgent)`
   *Point out: "We degraded the segment. The detection agent caught it, linear trend prediction determined it was critical, and the dispatcher automatically created a high-priority work order."*

### Phase 3: AI Telemetry Analysis (0:50 - 1:20)
1. **Inspect SEG-042:** Click on **SEG-042** in the grid.
2. **Observe Telemetry Panel:**
   - **Risk Gauge** is high.
   - **Vibration Chart** displays the progressive vibration history profile generated by the simulator.
   - **Trend Badge** alerts: `"Predicted critical in ~0 days"`.
   - **Ai Explanation** typewriter box dynamically generates a risk summary citing numbers, trends, and work orders using Gemini.

### Phase 4: Defect Verification & Closure (1:20 - 2:00)
1. **Verify Repair:**
   - In the verification section of the Telemetry Panel, paste a simulated repair description: 
     `"Tightened all loose rail bolts and recalibrated vibration sensors on segment 042."`
   - Click **Verify Repair**.
2. **Observe Result:**
   - **Activity Log** prints verification success and work order completion:
     `[VERIFICATION] VERIFY Repair verified for SEG-042 (confidence: X)`
     `[DISPATCH] WORK_ORDER Work order WO-SEG-042-001 completed — SEG-042 repaired`
   - SEG-042 returns to healthy green status on the grid.
   *Point out: "Gemini analyzed and verified the repair, automatically resolving the defects, resetting telemetry baseline, and closing the active work order with zero human intervention."*

## 🛠️ Dev Presets (Simulator Panel)

### Individual Simulation Actions (Target Segment)
- **Simulate Vibration Spike:** Instantly generates a progressive vibration spike history ramping from 2.0 mm/s to 9.5 mm/s on the target segment. This allows the linear trend prediction engine to register a rising trend and the chart to show a smooth curve.
- **Simulate Crack Detection:** Increments the segment's crack count by 1, simulating structural degradation.
- **Reset Segment:** Returns the selected segment back to its healthy baseline parameters and clears its defects.

### Demo Scenarios (Orchestrated Presets)
- **Degrade SEG-042 (Critical):** Automatically populates 20 progressive readings stepping from normal (2.0) to critical (8.5) vibration levels, increments crack counts, and fires work orders.
- **Mass Track Degradation:** Degrades 5 random segments to warning/critical status to show concurrent issues on the grid.
- **Reset All to Healthy:** Returns all 100 segments to healthy defaults and clears any pending work orders.
